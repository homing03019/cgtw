# 圖形格式 (RD)

`RD` 是 x-gate 用於儲存各類圖像資源（如角色、物件、背景等）的格式。通常包含索引檔 (`.info`) 與資料檔 (`.bin`)。

## 圖形索引 (GraphicInfo)

每個圖形的元數據佔用 **40 位元組**，結構如下：

- `ID` (int32): 圖形的唯一識別碼。
- `Addr` (int32): 資料在 `.bin` 檔案中的起始偏移量。
- `Len` (int32): 該圖形資料的長度。
- `OffX/OffY` (int32): 圖像渲染時的偏移量。
- `Width/Height` (int32): 圖像的尺寸。
- `GridW/GridH` (byte): 圖形佔用的網格單元數量。
- `Access` (byte): 存取標記（實作中目前未使用）。
- `_` (5 bytes): 填充/保留位元組。
- `MapID` (int32): 用於地圖關聯的 ID。

## 圖形資料 (GraphicHeader)

每個圖形資料塊的開頭通常包含一個標頭 (16 位元組)：

- `Magic`: 必須為 `"RD"` (2 bytes)。
- `Version`: 決定資料的儲存模式 (1 byte)：
  - `0`: 原始資料 (Raw)。
  - `1`: 經 RLE 編碼的資料。
  - `2`: 原始資料 + 內建調色盤。
  - `3`: 經 RLE 編碼的資料 + 內建調色盤。
- `_` (byte): 保留位元組。
- `Width/Height` (int32): 標頭中記錄的寬高（通常不可信，應以 `GraphicInfo` 為準）。
- `Len` (int32): 標頭中記錄的資料長度（通常不可信，應以 `GraphicInfo` 為準）。

## 渲染過程

1. **載入與解壓**: 若 `Version` 為 1 或 3，則調用 RLE 解碼演算法獲取索引資料。
2. **獲取調色盤**: 若版本不含調色盤，則載入全域 `.cgp` 檔案。
3. **映射顏色**: 將解碼後的每個位元組作為索引，從調色盤中查找對應的 RGBA 顏色值。
4. **輸出影像**: 將像素陣列轉換為標準的 `image.RGBA` 或 `image.Paletted` 物件。
