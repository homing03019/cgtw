# 動畫格式 (Anime)

動畫是 x-gate 用於呈現角色動作、法術特效等的資源格式。

## 動畫索引 (AnimeInfo)

每個動畫索引佔用 **12 位元組**：
- `ID` (AnimeID): 動畫的唯一識別碼。
- `Addr` (int32): 資料在動畫資料檔中的起始偏移量。
- `ActCnt` (int16): 該動畫包含的動作（Action）數量。
- `_` (int16): 填充/保留位元組。

## 動畫資料結構

動畫由多層級結構組成：

1. **動畫 (Anime)**: 對應一個 `AnimeID`。
2. **動作 (Action)**: 一個動畫可包含多個動作（如站立、跑步、攻擊、被擊中等）。
3. **畫格 (Frame)**: 每個動作由數個連續的畫格組成。

### 動畫標頭 (AnimeHeader)

每個動作都有一個標頭，長度可能為 **12 位元組** 或 **20 位元組 (v3)**：

- `Direct` (int16): 方向資訊。
- `Action` (ActionID): 動作類型 ID。
- `Duration` (int32): 動畫持續時間（毫秒）。
- `FrameCnt` (int32): 畫格數量。
- `_` (int16): 僅於 v3 出現的保留欄位。
- `Reversed` (int16): 僅於 v3 出現的保留欄位。
- `Sentinel` (int32): 僅於 v3 出現的哨兵值。若為 `-1`，則標頭長度判定為 20 位元組。

### 畫格資料 (AnimeFrameData)

每個畫格佔用 **10 位元組**：

- `GraphicID` (int32): 對應 `GraphicInfo` 中的 ID。
- `OffX/OffY` (int16): 渲染該畫格時的局部偏移。
- `Flag` (int16): 特殊標誌位（可用於標記攻擊判定、音效觸發等資訊）。

## 提取與轉換

`xgtool` 可以將這些動畫資料提取並轉換為 GIF：
1. **載入索引與標頭**: 定位動畫在資料檔中的位置，解析動作與畫格資訊。
2. **加載圖形資源**: 根據 `GraphicID` 讀取並解碼每一畫格對應的圖形。
3. **合成 GIF**: 將所有畫格按順序合成，並根據 `Duration` 設定每畫格的延遲時間。
